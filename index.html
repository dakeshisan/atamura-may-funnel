Путь клиента: реклама → лид → встреча → покупка

Май 2026. Сквозной путь 8 538 лидов телемаркетинга — от рекламной кампании, через две воронки обзвона и отдел продаж, до брони и оплаты. С разбивкой по каналам и ЖК.

Источник: Bitrix24 live (5 воронок: ТМ-инбокс · Zvandau · Отдел продаж · Клиентский сервис · Купившие) · история стадий + UTM по ID сделки · 78 704 события · 16.06.2026
Короткий ответ. Путь клиента лид → встреча → бронь прослеживается по ID сделки надёжно: сделка сохраняет свой ID и рекламную метку при переездах между воронками, и у 74–80% лидов есть источник/кампания. А вот последний шаг — покупку — к кампании привязать пока почти нельзя: покупки лежат в отдельной воронке без UTM, и цикл сделки длиннее месяца, поэтому из 21 сделки в воронке «Купившие» за май к майским лидам по ID цепляются лишь 3. Главный бизнес-вывод: 85% лидов отваливаются в телемаркетинге до встречи, а Meta Ads (68% объёма) даёт самую низкую конверсию в встречу. И теперь — со стоимостью: за май на рекламу ушло $32 тыс (Meta $26 + Google $6), встреча обходится ~$124–144 (Керуен/Aqsai — дешевле всех; Google дороже за лид, но качественнее).

Почему «кампания → покупка» пока неполная

1. Покупки — в отдельной воронке без рекламной метки. Воронка «Купившие» (по ЖК) заполнена UTM на 0%. Источник покупки можно узнать только протянув нить к лиду по ID.
2. Цикл сделки длиннее месяца. Из 21 сделки в воронке «Купившие» за май к майским лидам по ID цепляется 3 — остальные пришли из лидов прошлых месяцев. За один месяц атрибуция покупок неинформативна.
3. Пробелы в метках. У 20% лидов нет UTM-источника, у 26% — кампании, у 43% в метке не указан ЖК. Эти лиды выпадают из разреза.
Чтобы видеть полный путь «кампания → покупка» (приоритет):
  • Длинное окно наблюдения (квартал+): прослеживать покупки назад к лиду по ID за 3–6 мес, а не за месяц.
  • Дисциплина метки: проставлять ЖК и кампанию в UTM на 100% лидов; переносить метку в воронку «Купившие».
  • Свести с расходом Meta: добавить бюджет кампаний → получить стоимость лида (CPL) и стоимость встречи/покупки (CPA) по каналам и ЖК. Сейчас видно только конверсию, не деньги.

Оговорки

  • Стадия ≠ факт. «Встреча проведена», «Бронь» — отметки менеджера в CRM, не подтверждённые физически.
  • Диагностика, не KPI. Для понимания «где теряем», не для депремирования (риск по ТК РК).
  • Покупки по ID — нижняя граница. Сравнение каналов по покупкам делать осторожно (малые числа, длинный цикл).
  • Май, ТЗ портала (+03:00). Когорта = лиды, заведённые в мае, без дублей (исключено 1 462).
  • Нет расхода рекламы в данных Bitrix — стоимость лида/встречи не считается без выгрузки из Meta/Google.
Метод: когорта = сделки со стадией «новая заявка» (воронки Телемаркетинг/Zvandau) в мае, без ДУБЛЬ (8 538); прогресс = furthest milestone по ID через 5 воронок (монотонно ✓); источник/кампания = UTM на сделке, сохраняется при переездах.
